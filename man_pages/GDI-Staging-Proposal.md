## Chapter 4: Technical Architecture and Implementation

In this chapter, we delve into the technical aspects of the **Granular Data Integration (GDI) Staging Platform**, focusing on its architecture and implementation. We explore how this platform efficiently handles data, maintains referential integrity, ensures validation and review processes, and utilizes key technologies.

##### 4.1. System Architecture

The GDI Staging Platform is designed with a robust system architecture that enables efficient data processing, validation, and reporting. The architecture comprises the following key components:

- **Front-End Interface**: Utilizing **ReactJS**, a widely adopted and versatile front-end framework, the platform offers a user-friendly interface for data submission and reporting. Its popularity ensures accessibility to a broad talent pool for future development.

- **Back-End Engine**: The back-end is implemented in **Java** using the **Spring Framework**, providing a reliable and scalable foundation. This choice enables the use of an open-source **ISO 17442 compliant** library for generating and validating **Legal Entity Identifiers (LEIs)**, a crucial component of the platform's data standards.

- **Security Infrastructure**: The platform employs a robust security mechanism based on JSON Web Tokens (JWTs) with predefined expiry periods for each login session. This approach ensures the uniqueness and auditability of each login event, enhancing security and compliance.

- **Role-Based Access Control**: User roles are implemented to tailor the platform's user experience. Users are presented with data relevant to their specific domain, preventing information overload and ensuring a focused user interface.

- **Elasticsearch**: This search engine plays a vital role in maintaining referential integrity by orchestrating data searches. It assists engineers in populating React forms with necessary reference data during data entry.

- **Jasper Reports**: The platform leverages the **Jasper Reports library** to generate reports directly from queries. This feature simplifies the report creation process, enabling collaboration with Database Administrators (DBAs) to create custom reports from the front end, without the need for backend code modifications. These reports are instrumental in conducting comparative analyses with aggregate reports, a crucial step before compiling granular data for submission to the regulator.

##### 4.2. Data Structures

The data within the GDI Staging Platform can be categorized into three main types:

- **Master Data**: This data, provided by the regulator, specifies items used in the representative data fields for submission. It acts as a reference for the information collected.

- **Metadata**: Metadata refers to data about the data entities within the representative data itself. It includes information necessary for validation, reporting, and ensuring data integrity.

- **Representative Data**: This is the core dataset representing the organization's economic status, financial performance, risk parameters, and other essential data points. It forms the basis of reporting to regulatory authorities.

This categorization helps streamline data management and facilitates efficient data processing.

The technical architecture of the GDI Staging Platform offers several benefits:

- **Legal Entity Identifier (LEI) Compliance**: The platform's Java-based backend ensures compliance with LEI standards, even before they become mandatory. It maintains data privacy and confidentiality while aligning with international trends.

- **Referential Integrity**: By design, the platform maintains referential integrity, reducing the challenges associated with manual data reconciliation.

- **Validation and Review**: The system enforces validation rules and provides a platform for quick and accurate data reviews, enhancing data quality.

- **Collaboration**: The user-friendly front-end interface promotes collaboration among stakeholders. Users can contribute to the platform from anywhere, facilitating faster data submissions and insights.
